When designing your web applications,

one must always use input validations,

for the user may be good or be bad,

and the worst is losing the data you had.

If your database happens to be relatoional,

then protecting against SQL injections is foundational,

for if the user inputs DROP DATABASE *,

you better sanitize that input before it gets too far.

Do not forget SQL instances run on a port,

so your SQL API should have defense like a fort,

filter incoming traffic based on network addresses,

and make changes to built-in accounts to improve your defenses.

Whatever format your database is in,

you must protect the data that is contained within,

for that is the mission of the database administrator,

to protect against the ill-intent of network penetrators.

-Mike Berg
